# Database Fix for Num1Store

## Issue
The products page is showing an error because the database schema doesn't match what the code expects:
- `products.category` column doesn't exist
- `categories` table doesn't exist or has wrong schema

## Quick Fix

### Step 1: Run the Database Fix
1. Open your Supabase Dashboard
2. Go to the SQL Editor
3. Copy the contents of `database-fix.sql` (generated by running `node fix-database-issues.js`)
4. Paste and run the SQL script

### Step 2: Verify the Fix
Run the test script to verify everything works:
```bash
node test-products.js
```

### Step 3: Check the Website
Visit http://localhost:3000/products - the error should be resolved and you should see products.

## What the Fix Does

### Products Table
- Adds missing columns: `category`, `is_active`, `inventory_count`, `images`, `vendor_id`
- Updates existing products with appropriate categories
- Fixes Row Level Security (RLS) policies

### Categories Table  
- Creates the categories table with proper schema
- Inserts sample categories (Electronics, Fashion, Sports, etc.)
- Sets up RLS policies for public read access

## Manual SQL Commands

If you prefer to run commands manually, here are the key fixes:

```sql
-- Add missing columns to products
ALTER TABLE products ADD COLUMN IF NOT EXISTS category TEXT NOT NULL DEFAULT 'Uncategorized';
ALTER TABLE products ADD COLUMN IF NOT EXISTS is_active BOOLEAN DEFAULT TRUE;
ALTER TABLE products ADD COLUMN IF NOT EXISTS inventory_count INTEGER DEFAULT 0;
ALTER TABLE products ADD COLUMN IF NOT EXISTS images TEXT[] DEFAULT '{}';
ALTER TABLE products ADD COLUMN IF NOT EXISTS vendor_id UUID REFERENCES vendors(id);

-- Update existing products
UPDATE products SET 
    category = CASE 
        WHEN name ILIKE '%headphone%' OR name ILIKE '%watch%' THEN 'Electronics'
        WHEN name ILIKE '%coffee%' THEN 'Food'
        WHEN name ILIKE '%yoga%' OR name ILIKE '%shoe%' THEN 'Sports'
        ELSE 'Uncategorized'
    END,
    is_active = COALESCE(is_active, TRUE),
    inventory_count = COALESCE(inventory_count, 50),
    images = COALESCE(images, ARRAY['/images/placeholder.jpg']);
```

## After the Fix

Once you run the database fix:
- The products page should load without errors
- You should see product listings with categories
- The category filter should work properly
- Search and sorting functionality should work

## Troubleshooting

If you still see errors:
1. Check the browser console for specific error messages
2. Verify the SQL script ran successfully in Supabase
3. Make sure your `.env.local` has the correct Supabase credentials
4. Run `node test-products.js` to verify database connectivity
